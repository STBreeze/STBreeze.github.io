Project Overview

This project focuses on developing a predictive maintenance model for the NASA Turbofan Engine Degradation Dataset. The goal is to predict the Remaining Useful Life (RUL) of turbofan engines. By doing so, maintenance strategies can be optimized, reducing costs and preventing unexpected failures. The model uses a hybrid combination of regressive prediction with time-series features incorporated to predict the RUL. This is because traditional SARIMAX models were not suitable due to the inconsistent engine lifecycle trends.

Problem Statement

There is no denying that engine failures can be catastrophic leading to costly maintenance, operational disruptions and safety risks. Therefore, prediciting an engine's remaining useful life or RUL is crucial for preventive maintenance and reducing unexpected failures. For this project, NASA put out an open challenge of their CMAPSS dataset with the goal to develop a predictive model to estimate the degredation of their turbofan engines overtime. Through these predictions, proactive maintenance measures can be implemented, saving costs and improving reliability in aerospace operations.

Goals & Challenges

I initially pursued this project to expand on my IBM Data Science course and build my portfolio. However, it introduced me to Time-Series analysis, a concept I was unfamiliar with. Over the following weeks, I explored time-series methods, initially considering a SARIMAX model to solve this problem by treating the number of engines as the seasonal component, engine cycles as the time component, and sensors with operational settings as exogenous factors. However, due to inconsistent engine lifecycles, this approach proved unsuitable. Instead, I opted for a RandomForest regression model trained on datasets that incorporated time-series features like moving averages and consecutive differences. This allowed me to avoid the complexity of deep learning models like CNNs or LSTMs while maintaining accuracy.

Dataset Description

The dataset consists of three files:

  • Training Data (train_FD001.txt): Contains run-to-failure data for 100 engines.
  • Test Data (test_FD001.txt): Contains operational data for 100 engines, stopped before failure.
  • RUL Data (RUL_FD001.txt): Contains the actual RUL values for the test engines at the point of failure.

Each engine is monitored using 21 sensors and 3 operational settings, recorded over multiple cycles. The dataset includes noise and anomalies, which were addressed during preprocessing.

Methodology

The project followed the following steps:

  • Static columns (e.g., OpSet_3, Sensor_1, etc.) were removed to reduce dimensionality.
  • Anomalies were detected and replaced using polynomial regression and percentile-based methods.
  • Time-series features like moving averages and consecutive differences were introduced to capture trends.
  • A Random Forest Regressor was used to predict RUL.
  • The model was trained on the preprocessed training data and evaluated on the test set.
  • Performance metrics like MSE, RMSE, MAE, MAPE, and R² were calculated.
  • Predicted RUL values were compared against actual RUL values.

This graph visualizes the lifecycle of all engines, color-coded across 3 operational settings and 21 sensors of the training set. It highlights global degradation trends, with some sensors remaining constant (suitable for omission) and others showing accelerating, decelerating, or mixed patterns. Sensors 9 & 14 exhibit mixed trends, while Operational Setting 2 has discrete values.

The two images highlight the before & after stages of the anomaly handling process. Using a percentile-based method, 5% of the extreme ends of the data was taken as anomalies. These anomalies were determined by taking the difference between the predicted datapoints, from the polynomial regression line, and the actual datapoints. Upon identifying the anomalies, they were replaced by the predicted datapoints, as dropping them would reduce the overall data quantity.

Results

  • Mean Squared Error (MSE): 661.31
  • Mean Squared Error (RMSE): 25.72
  • Absolute Error (MAE): 19.41
  • Absolute Percentage Error (MAPE): 0.31
  • R² Score: 0.62

Conclusion

The predictive maintenance model successfully forecasts the RUL of turbofan engines with reasonable accuracy. The use of time-series features and robust anomaly handling improved the model's performance. However, there is room for improvement in handling noise and capturing complex trends.
